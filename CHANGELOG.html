0.1.11 — 2026-03-17

  • Fixed: object-fit and object-position not applying to the scrolled logo — removed fragile JS inline assignment that read from data-settings (unreliable when value equals the control default); replaced with Elementor selectors output targeting .mk-em-logo-scrolled directly; CSS defaults in sticky-header.css cover the case where Elementor skips outputting the default value.

0.1.10 — 2026-03-17

  • Fixed: object-position not reflected on frontend — replaced stale module-level IS_EDITOR constant (evaluated before Elementor's frontend was ready) with the documented $(window).on('elementor/frontend/init') + elementorFrontend.hooks.addAction('frontend/element_ready/global') pattern; isEditor is now resolved reliably inside the callback and passed through the entire call chain.
  • Fixed: Editor preview switch had no effect / caused a flash — same root cause; the editor branch of init() never ran because isEditor was always false at module load time.
  • Fixed: Settings-change MutationObserver re-init used stale isEditor = false; now captures the correct value in its closure.

0.1.9 — 2026-03-17

  • Fixed: object-position (and any render_type:none control) not updating live — a per-element MutationObserver on data-settings now re-inits the handler on every control change in the editor.
  • Fixed: Editor detection uses three fallbacks: elementor-editor-preview body class, elementorFrontend.isEditMode(), and window.parent.elementor parent-frame check.
  • Fixed: Preview switch no longer causes a flash — changed to render_type:none, handled entirely by the attribute observer.

0.1.8 — 2026-03-17

  • Fixed: object-fit still not applied — Elementor also skips serialising frontend_available values equal to the default; fit and position are now always written with explicit fallbacks.
  • Fixed: Editor detection was unreliable — added elementor-frontend as a script dependency so our code runs after Elementor's frontend is ready.
  • Added: Preview Scrolled State toggle — turn on in the editor to see background, shadow, logo swap and height exactly as they appear after scrolling.

0.1.7 — 2026-03-17

  • Fixed: Scrolled Logo Fit had no effect — object-fit and object-position are now applied as inline styles by JS instead of relying on Elementor's CSS selector output (which skips default values).
  • Fixed: Editor preview showed a normal container regardless of settings — JS now detects edit mode and renders the sticky scrolled state as a static preview; a MutationObserver re-initialises on every settings change re-render.

0.1.6 — 2026-03-17

  • Fixed: Container with background image only (no content widgets) was invisible — height fallback now uses at least 80px.
  • Changed: Background (scrolled) now supports solid color, gradient, and image via Elementor group control.
  • Changed: Box Shadow (scrolled) is now fully editable (color, blur, spread, offset, inset) instead of on/off toggle.
  • Added: Scrolled Logo Position — 9-point anchor selector for the alternate logo image.

0.1.5 — 2026-03-17

  • Fixed: Height shrink/expand had no animation — all transitions are now set as inline styles by JS (reads Transition Duration setting), bypassing Elementor's own transition on containers.
  • Fixed: Transition Duration control now marked frontend_available so JS can read the configured value.

  • Fixed: Logo scale animation snapping back instead of animating when scrolling up — transition is now declared on the base state.
